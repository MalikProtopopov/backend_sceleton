# Требования к фронтенду админки: биллинг и тарифы

> Что добавить / изменить в админ-панели для работы с новым биллинг-API

---

## 1. Область видимости

| Роль | Что видит |
|------|-----------|
| **Любой авторизованный** (site_owner, content_manager, editor) | «Мой тариф», «Мои модули», «Лимиты», каталог тарифов, заявки на апгрейд |
| **platform_owner / superuser** | Всё выше + управление планами, модулями, бандлами, заявками, модулями тенантов |

---

## 2. Новые страницы и разделы

### 2.1. Раздел «Биллинг» / «Тариф» (для всех пользователей)

**Маршрут:** `/admin/billing` или `/admin/my-plan`

**Содержимое:**
- Карточка текущего плана (название, описание, цены)
- Блок «Использование ресурсов» — прогресс-бары по лимитам
- Список активных модулей
- Кнопка «Сменить тариф» → переход к каталогу планов

**API:** `GET /api/v1/admin/my-plan`

---

### 2.2. Страница «Мои модули»

**Маршрут:** `/admin/billing/modules` или вкладка в разделе «Биллинг»

**Содержимое:** Таблица активных модулей (название, источник, дата активации, статус).

**API:** `GET /api/v1/admin/my-modules`

---

### 2.3. Страница «Лимиты»

**Маршрут:** `/admin/billing/limits` или вкладка в «Биллинг»

**Содержимое:** Прогресс-бары по каждому лимиту (пользователи, хранилище, заявки, товары, статьи, домены).

**API:** `GET /api/v1/admin/my-limits`

---

### 2.4. Каталог тарифов (для выбора / сравнения)

**Маршрут:** `/admin/billing/plans` или модальное окно

**Содержимое:**
- Таблица сравнения планов (цены, модули, лимиты)
- Выделение текущего плана
- Кнопки «Выбрать» / «Оставить заявку» для каждого плана

**API:** `GET /api/v1/public/plans`, `GET /api/v1/public/modules`, `GET /api/v1/public/bundles`

---

### 2.5. Заявки на апгрейд

**Маршрут:** `/admin/billing/requests`

**Содержимое:**
- Список своих заявок (дата, тип, цель, статус)
- Кнопка «Оставить заявку» → модальное окно с формой

**API:** `GET /api/v1/admin/upgrade-requests`, `POST /api/v1/admin/upgrade-requests`

---

### 2.6. Platform: управление планами (только platform_owner)

**Маршрут:** `/admin/platform/plans`

**Содержимое:** CRUD планов (список, создание, редактирование).

**API:** `GET/POST/PATCH /api/v1/admin/platform/plans`

---

### 2.7. Platform: управление модулями

**Маршрут:** `/admin/platform/modules`

**API:** `GET/POST/PATCH /api/v1/admin/platform/modules`

---

### 2.8. Platform: управление бандлами

**Маршрут:** `/admin/platform/bundles`

**API:** `GET/POST/PATCH /api/v1/admin/platform/bundles`

---

### 2.9. Platform: обработка заявок

**Маршрут:** `/admin/platform/requests`

**Содержимое:** Таблица заявок с фильтром по статусу, кнопки «Одобрить» / «Отклонить».

**API:** `GET /api/v1/admin/platform/upgrade-requests`, `PATCH /api/v1/admin/platform/upgrade-requests/{id}`

---

### 2.10. Platform: модули тенанта (в карточке тенанта)

**Маршрут:** Вкладка «Биллинг-модули» в карточке редактирования тенанта (например, `/admin/platform/tenants/{id}`).

**Содержимое:**
- **Таблица активных модулей** — загрузить через `GET /api/v1/admin/platform/tenants/{tenant_id}/modules`
- **Блок «Добавить модуль вручную»** — дропдаун + кнопка «Добавить»
- **Блок «Удалить модуль»** — дропдаун + кнопка «Удалить»

**API:**
- `GET /api/v1/admin/platform/tenants/{tenant_id}/modules` — список активных модулей
- `GET /api/v1/admin/platform/modules` — все модули (для дропдаунов)
- `POST /api/v1/admin/platform/tenants/{tenant_id}/modules` — добавить
- `DELETE /api/v1/admin/platform/tenants/{tenant_id}/modules` — удалить

## 3. Обязательные изменения в существующих экранах

### 3.1. Вкладка «Биллинг-модули» в карточке тенанта

**Проблема:** Дропдаун «Добавить модуль» показывает все модули, включая уже активные → 409 при добавлении.

**Требование:**
1. При загрузке вкладки вызывать `GET /admin/platform/tenants/{tenant_id}/modules` и `GET /admin/platform/modules`.
2. В дропдауне «Добавить модуль» показывать **только модули, которых нет** в списке активных.
3. В дропдауне «Удалить модуль» показывать **только активные** модули тенанта.
4. При 409 всё равно показывать понятное сообщение: «Модуль уже подключён».

**Логика:**
```typescript
const activeSlugs = new Set(activeModules.map(m => m.module_slug))
const availableToAdd = allModules.filter(m => !activeSlugs.has(m.slug))
const availableToRemove = activeModules
```

---

### 3.2. Обработка ошибки лимита (403 limit_exceeded)

При создании статьи, продукта, пользователя бэкенд может вернуть 403 с `error_code: "limit_exceeded"`.

**Требование:** Перехватывать такую ошибку и показывать:
- Тост/баннер: «Лимит исчерпан: {ресурс}. Использовано {current} из {limit}.»
- Кнопку «Обновить тариф» → переход на `/admin/billing/plans`.

---

### 3.3. Обработка rate limit (429) при заявках

При `POST /admin/upgrade-requests` возможен 429 (5 заявок в час).

**Требование:** Показать сообщение: «Слишком много заявок. Попробуйте позже.» и не давать отправить форму повторно до истечения `retry_after`.

---

## 4. Утилиты и маппинги

### 4.1. Форматирование цен

Цены приходят в **копейках** (int). Отображать: `(kopecks / 100).toLocaleString('ru-RU', { style: 'currency', currency: 'RUB', minimumFractionDigits: 0 })`.

### 4.2. Лимиты

| Значение | Отображение |
|----------|-------------|
| `-1` | «∞» или «Без ограничений» |
| `0` | «—» или «Не включено» |
| `> 0` | Число (для `max_storage_mb` — перевести в ГБ при ≥ 1024) |

### 4.3. Источник модуля (source)

| source | Метка (RU) |
|--------|------------|
| `plan` | Из тарифа |
| `addon` | Допокупка |
| `bundle` | Из пакета |
| `manual` | Вручную |

### 4.4. Статус заявки

| status | Метка (RU) | Цвет бейджа |
|--------|------------|--------------|
| `pending` | На рассмотрении | Жёлтый |
| `approved` | Одобрена | Зелёный |
| `rejected` | Отклонена | Красный |

### 4.5. Тип заявки

| request_type | Метка (RU) |
|--------------|------------|
| `plan_upgrade` | Смена тарифа |
| `module_addon` | Покупка модуля |
| `bundle_addon` | Покупка пакета |

### 4.6. Прогресс-бар по status

| status | Цвет |
|--------|------|
| `ok` | Зелёный / нейтральный |
| `warning` | Оранжевый |
| `exceeded` | Красный |

## 6. Тело запросов

### POST /admin/upgrade-requests

```json
{
  "request_type": "plan_upgrade",
  "target_plan_id": "uuid",
  "message": "Комментарий (необязательно)"
}
```

Варианты: `plan_upgrade` + `target_plan_id`, `module_addon` + `target_module_id`, `bundle_addon` + `target_bundle_id`.

### POST /admin/platform/tenants/{id}/modules

```json
{
  "module_slug": "seo_advanced",
  "source": "manual",
  "enabled": true
}
```

### DELETE /admin/platform/tenants/{id}/modules

```json
{
  "module_slug": "seo_advanced"
}
```
