# Промпты для генерации контента

Промпты для AI-ассистента для генерации структурированного контента на основе предоставленного контекста о компании.

---

## Как использовать

1. Скопируйте промпт для нужного типа контента
2. Замените `[КОНТЕКСТ]` на информацию о вашей компании/проекте
3. Отправьте AI-ассистенту
4. Получите структурированные данные для заполнения админ-панели

---

## 1. Услуги (Services)

### Промпт для генерации услуг

```
Ты — опытный контент-менеджер. На основе контекста о компании создай структурированное описание услуги для сайта.

## Контекст о компании:
[КОНТЕКСТ]

## Услуга для описания:
[НАЗВАНИЕ УСЛУГИ]

## Требования к формату:

Верни данные в следующей структуре (JSON):

{
  "service": {
    "icon": "название-иконки (например: code, chart-line, users)",
    "price_from": число или null,
    "price_currency": "RUB",
    "locales": {
      "ru": {
        "title": "Название услуги (до 255 символов)",
        "slug": "url-slug-latinicey (до 255 символов)",
        "short_description": "Краткое описание для карточки (до 500 символов)",
        "description": "Полное описание услуги в формате Markdown (без лимита)",
        "meta_title": "SEO заголовок (до 70 символов)",
        "meta_description": "SEO описание (до 160 символов)"
      },
      "en": {
        "title": "Service Title",
        "slug": "service-slug",
        "short_description": "Short description",
        "description": "Full description in Markdown",
        "meta_title": "SEO Title",
        "meta_description": "SEO Description"
      }
    },
    "tags": ["React", "TypeScript", "Node.js"]
  }
}

## Правила:
1. slug должен быть на латинице, в нижнем регистре, слова через дефис
2. meta_title — максимум 70 символов, включает ключевое слово и бренд
3. meta_description — максимум 160 символов, призыв к действию
4. short_description — ёмко описывает выгоду для клиента
5. description — структурировано с заголовками h2/h3, списками, выделением
6. Теги — технологии, инструменты, подходы (3-7 штук)
```

## 2. Статьи / Блог (Articles)

### Промпт для генерации статьи

```
Ты — опытный контент-маркетолог. На основе контекста создай статью для корпоративного блога.

## Контекст о компании:
[КОНТЕКСТ]

## Тема статьи:
[ТЕМА]

## Целевая аудитория:
[ОПИСАНИЕ АУДИТОРИИ]

## Требования к формату:

{
  "article": {
    "status": "draft",
    "reading_time_minutes": число,
    "locales": {
      "ru": {
        "title": "Заголовок статьи (до 255 символов)",
        "slug": "url-slug-latinicey",
        "excerpt": "Анонс/лид статьи (до 500 символов)",
        "content": "Полный текст статьи в Markdown",
        "meta_title": "SEO заголовок (до 70 символов)",
        "meta_description": "SEO описание (до 160 символов)"
      }
    },
    "topics": ["Название категории 1", "Название категории 2"]
  }
}

## Правила:
1. Заголовок — цепляющий, содержит ключевое слово
2. Excerpt — интригует, мотивирует читать дальше (до 500 символов)
3. Content — структура: введение, 3-5 разделов с h2, заключение с CTA
4. Время чтения — рассчитай исходя из ~200 слов/минуту
5. meta_title — включи ключевое слово в начале
6. meta_description — содержит ключевое слово и призыв к действию
7. Используй списки, цитаты, выделение ключевых мыслей
```

## 3. Кейсы / Портфолио (Cases)

### Промпт для генерации кейса

```
Ты — опытный копирайтер, специализирующийся на B2B кейсах. Создай описание кейса/проекта.

## Контекст о компании:
[КОНТЕКСТ]

## Информация о проекте:
- Клиент: [НАЗВАНИЕ КЛИЕНТА или "конфиденциально"]
- Отрасль: [ОТРАСЛЬ]
- Задача: [ОПИСАНИЕ ЗАДАЧИ]
- Решение: [ЧТО СДЕЛАЛИ]
- Результаты: [МЕТРИКИ, ЦИФРЫ]
- Год: [ГОД]
- Длительность: [СРОК]

## Требования к формату:

{
  "case": {
    "status": "published",
    "client_name": "Название клиента или null",
    "project_year": 2024,
    "project_duration": "3 месяца",
    "is_featured": true/false,
    "services": ["Связанная услуга 1", "Связанная услуга 2"],
    "locales": {
      "ru": {
        "title": "Название кейса (до 255 символов)",
        "slug": "url-slug-latinicey",
        "excerpt": "Краткое описание для карточки (до 500 символов)",
        "description": "Полное описание проекта в Markdown",
        "results": "Результаты проекта в Markdown (метрики, достижения)",
        "meta_title": "SEO заголовок (до 70 символов)",
        "meta_description": "SEO описание (до 160 символов)"
      }
    }
  }
}

## Правила для description:
1. Структура: Задача → Вызовы → Решение → Процесс → Технологии
2. Используй конкретные детали, избегай общих фраз
3. Включи технические подробности для целевой аудитории

## Правила для results:
1. Конкретные цифры и метрики
2. До/После сравнение
3. Бизнес-результаты (ROI, экономия, рост)
4. Технические улучшения (скорость, надёжность)

## Пример results:
"## Результаты проекта

- **+150%** рост конверсии сайта
- **-40%** время загрузки страниц
- **99.9%** uptime за 12 месяцев
- Экономия **500 000 ₽/мес** на инфраструктуре"
```

## 4. Отзывы (Reviews)

### Промпт для структурирования отзывов

```
Ты — контент-менеджер. Структурируй отзыв клиента для сайта.

## Исходный отзыв:
[ТЕКСТ ОТЗЫВА]

## Информация об авторе:
- Имя: [ИМЯ]
- Компания: [КОМПАНИЯ]
- Должность: [ДОЛЖНОСТЬ]
- Источник: [google/clutch/yandex/direct/etc]

## Требования к формату:

{
  "review": {
    "status": "approved",
    "rating": 5,
    "author_name": "Имя Фамилия (2-255 символов)",
    "author_company": "Название компании или null",
    "author_position": "Должность или null",
    "content": "Текст отзыва (минимум 10 символов)",
    "source": "google/clutch/yandex/direct",
    "source_url": "URL оригинала или null",
    "review_date": "2024-01-15",
    "is_featured": true/false,
    "case_id": "ID связанного кейса или null"
  }
}

## Правила:
1. author_name — минимум 2 символа
2. content — минимум 10 символов
3. rating — от 1 до 5
4. Если отзыв слишком длинный — сократи до ключевых мыслей
5. Сохрани живой язык и эмоции автора
6. is_featured = true для лучших отзывов (с конкретикой и цифрами)
```

## 5. FAQ (Часто задаваемые вопросы)

### Промпт для генерации FAQ

```
Ты — контент-менеджер. Создай FAQ для сайта компании.

## Контекст о компании:
[КОНТЕКСТ]

## Категория FAQ:
[КАТЕГОРИЯ: Общие / Оплата / Доставка / Услуги / и т.д.]

## Количество вопросов:
[ЧИСЛО]

## Требования к формату:

{
  "faqs": [
    {
      "category": "Общие",
      "is_published": true,
      "sort_order": 0,
      "locales": {
        "ru": {
          "question": "Вопрос? (5-500 символов)",
          "answer": "Ответ в Markdown (без лимита)"
        },
        "en": {
          "question": "Question?",
          "answer": "Answer in Markdown"
        }
      }
    }
  ]
}

## Правила:
1. question — минимум 5 символов, максимум 500
2. Формулируй вопросы от лица клиента (как он спросил бы)
3. Ответы — конкретные, полезные, с примерами где уместно
4. Используй форматирование в ответах (списки, выделение)
5. Включи вопросы которые снимают возражения клиентов
6. sort_order — от самых частых (0) к редким

## Типичные категории:
- Общие вопросы о компании
- Услуги и цены
- Процесс работы
- Сроки и гарантии
- Оплата
- Техническая поддержка
```

```
- Должность: [ДОЛЖНОСТЬ]
- Опыт: [ОПЫТ И ДОСТИЖЕНИЯ]
- Контакты: [EMAIL, ТЕЛЕФОН — опционально]

## Требования к формату:

{
  "employee": {
    "email": "email@company.com или null",
    "phone": "+7 999 123-45-67 или null",
    "linkedin_url": "URL или null",
    "telegram_url": "https://t.me/username или null",
    "is_published": true,
    "sort_order": 0,
    "locales": {
      "ru": {
        "first_name": "Имя (до 100 символов)",
        "last_name": "Фамилия (до 100 символов)",
        "slug": "imya-familiya",
        "position": "Должность (до 255 символов)",
        "bio": "Биография в Markdown",
        "meta_title": "Имя Фамилия — Должность | Компания (до 70 символов)",
        "meta_description": "Краткое описание для поисковиков (до 160 символов)"
      },
      "en": {
        "first_name": "First Name",
        "last_name": "Last Name",
        "slug": "first-last",
        "position": "Position",
        "bio": "Bio in Markdown",
        "meta_title": "Name — Position | Company",
        "meta_description": "Short description"
      }

  }
}

## Правила для bio:
1. Начни с текущей роли и экспертизы
2. Упомяни ключевые достижения и проекты
3. Добавь профессиональные интересы
4. Объём: 100-300 слов
5. Тон: профессиональный, но дружелюбный

## Пример bio:
"Руководит разработкой продуктов компании с 2019 года. За это время под его руководством выпущено более 50 проектов для клиентов из финтеха, e-commerce и логистики.

**Экспертиза:**
- Архитектура высоконагруженных систем
- Управление распределёнными командами
- Agile/Scrum методологии

Ранее работал в Яндексе и Mail.ru Group. Спикер конференций HighLoad++ и TeamLead Conf."
```

## 7. SEO-настройки страниц

### Промпт для генерации SEO-данных

```
Ты — SEO-специалист. Создай SEO-настройки для страницы сайта.

## Контекст о компании:
[КОНТЕКСТ]

## Страница:
- URL: [ПУТЬ, например /services/web-development]
- Тип: [Главная / Услуга / Кейс / Статья / Категория]
- Основное ключевое слово: [КЛЮЧЕВОЕ СЛОВО]

## Требования к формату:

{
  "seo_route": {
    "path": "/services/web-development",
    "locale": "ru",
    "title": "Заголовок страницы (до 70 символов)",
    "meta_title": "SEO Title — Бренд (до 70 символов)",
    "meta_description": "SEO описание с ключевым словом и CTA (до 160 символов)",
    "canonical_url": "https://example.com/services/web-development или null",
    "robots_index": true,
    "robots_follow": true,
    "sitemap_priority": 0.8,
    "sitemap_changefreq": "weekly",
    "include_in_sitemap": true,
    "structured_data": "JSON-LD разметка"
  }
}

## Правила:
1. meta_title: ключевое слово в начале, бренд в конце, до 70 символов
2. meta_description: ключевое слово, выгода, призыв к действию, до 160 символов
3. sitemap_priority: главная=1.0, услуги=0.8, кейсы=0.7, статьи=0.6, FAQ=0.5
4. sitemap_changefreq: статические=monthly, блог=weekly, главная=daily

## Приоритеты страниц:
- / (главная): 1.0, daily
- /services: 0.9, weekly
- /services/*: 0.8, monthly
- /cases: 0.8, weekly
- /cases/*: 0.7, monthly
- /blog: 0.7, daily
- /blog/*: 0.6, weekly
- /about, /contacts: 0.5, monthly
- /faq: 0.5, monthly
- /privacy, /terms: 0.3, yearly
```

## 8. Массовая генерация контента

### Промпт для генерации всего контента сайта

```
Ты — контент-директор. На основе информации о компании создай полный набор контента для сайта.

## Информация о компании:
[ПОЛНЫЙ КОНТЕКСТ: название, сфера деятельности, услуги, преимущества, целевая аудитория, tone of voice]

## Что нужно создать:

1. **Услуги** (5-10 штук):
   - Название, slug, описания, теги
   - SEO-данные

2. **FAQ** (10-15 вопросов):
   - Общие вопросы о компании
   - Вопросы по услугам
   - Вопросы по процессу работы

3. **SEO для основных страниц**:
   - Главная (/)
   - О компании (/about)
   - Услуги (/services)
   - Контакты (/contacts)

## Формат ответа:
Верни JSON с разделами: services, faqs, seo_routes

## Tone of Voice:
[ОПИСАНИЕ ТОНА: профессиональный/дружелюбный/экспертный/и т.д.]

## Язык:
Русский (основной), English (опционально)
```

## Примеры использования

### Пример 1: Генерация услуги для IT-компании

**Контекст:**
```
Компания "ДевСтудио" — веб-студия из Москвы. Разрабатываем сайты и веб-приложения на React, Node.js, Python. Клиенты — средний и крупный бизнес. Tone of voice — экспертный, но понятный.
```

**Услуга для описания:**
```
Разработка корпоративных порталов
```

### Пример 2: Генерация FAQ для SaaS-продукта

**Контекст:**
```
CRM-система "СмартСейлс" для отделов продаж. Облачное решение, подписка от 990 руб/мес. Интеграции с телефонией, почтой, мессенджерами. Целевая аудитория — малый и средний бизнес.
```

**Категории FAQ:**
```
Общие, Тарифы и оплата, Интеграции, Техподдержка
```

## Чек-лист качества контента

После генерации проверьте:

- [ ] Все slug на латинице, без пробелов, в нижнем регистре
- [ ] meta_title ≤ 70 символов
- [ ] meta_description ≤ 160 символов
- [ ] short_description / excerpt ≤ 500 символов
- [ ] Нет орфографических ошибок
- [ ] Тексты уникальны, не скопированы
- [ ] Присутствуют ключевые слова
- [ ] CTA (призыв к действию) в описаниях
- [ ] Структура текстов логична (заголовки, списки)
- [ ] Tone of voice соответствует бренду
