# Коммерческое предложение: Платформа управления корпоративным сайтом

> **Версия:** 1.0  
> **Дата:** 13 февраля 2026  
> **Документ:** Оценка стоимости серверной части (бэкенд) с разбивкой по модулям

---

## 1. Резюме: что делает система

Платформа представляет собой **систему управления корпоративным сайтом** с возможностью обслуживания нескольких организаций из единого центра. Каждая организация получает собственную изолированную рабочую среду с настраиваемым набором возможностей.

### Какие бизнес-процессы закрывает:

- **Публикация контента** — статьи, новости, кейсы, документы, FAQ, отзывы
- **Презентация компании** — услуги, команда, направления работы, преимущества, контакты
- **Работа с обращениями** — приём заявок, статусы обработки, назначение ответственного, аналитика источников
- **Управление командой администраторов** — роли, доступы, разграничение прав
- **Поисковая оптимизация** — управление мета-тегами, карты сайта, перенаправления, интеграция с поисковиками
- **Уведомления** — отправка писем и сообщений в Telegram при событиях
- **Отчётность** — сводная панель, экспорт данных, журнал действий
- **Обслуживание нескольких организаций** — каждому клиенту свой набор данных, домен и настройки

### Кто пользователи:

| Роль | Описание |
|------|----------|
| **Посетитель сайта** | Читает контент, отправляет заявки, ищет специалистов |
| **Администратор организации** | Управляет контентом, обрабатывает заявки, смотрит аналитику |
| **Контент-менеджер** | Публикует статьи, кейсы, обновляет информацию о компании |
| **Маркетолог** | Работает с заявками, UTM-аналитикой, настраивает формы |
| **Владелец платформы** | Управляет организациями, подключает/отключает модули, видит общую картину |

## 2. Модули и функции (продуктовые блоки)

### 2.1. Управление организациями

Единый центр для работы с несколькими клиентами. Каждая организация — изолированная среда со своими данными, доменом, настройками и набором включённых функций.

**Возможности:**
- Создание и настройка организаций (название, домен, логотип, фирменные цвета)
- Включение/отключение модулей для каждой организации (блог, кейсы, отзывы, FAQ, команда, услуги)
- Привязка собственных доменов для панели управления
- Настройка часовых поясов, форматов дат, языков
- Переключение между организациями одним действием (если у сотрудника несколько организаций)

### 2.2. Пользователи, роли и доступы

Гибкая система прав: кто что видит, кто что может делать.

**Возможности:**
- Регистрация и приглашение новых администраторов
- Настраиваемые роли (админ, редактор, маркетолог, зритель)
- Разграничение прав: кто может публиковать, кто только редактировать
- Личные профили с аватаром
- Безопасный вход с обновляемыми ключами доступа
- Восстановление пароля через электронную почту
- Журнал всех действий (кто, когда, что изменил)

### 2.3. Управление контентом (блог)

Полноценная система публикаций: статьи, рубрики, черновики, публикация по кнопке.

**Возможности:**
- Создание статей с обложкой, описанием, полным текстом
- Рубрики (темы) для группировки статей
- Статусы: черновик → опубликовано → в архиве
- Подсчёт просмотров и времени чтения
- Гибкие блоки контента: текст, изображения, видео, галереи, ссылки
- Поддержка нескольких языков

### 2.4. Портфолио / Кейсы

Демонстрация выполненных проектов с результатами.

**Возможности:**
- Описание проекта: клиент, год, длительность, результаты
- Привязка кейсов к конкретным услугам
- Контактные данные проекта
- Статусы публикации
- Избранные кейсы для главной страницы

### 2.5. Отзывы и рекомендации

Управление отзывами клиентов с модерацией.

**Возможности:**
- Добавление отзывов: автор, компания, должность, рейтинг (1–5), текст
- Привязка отзыва к конкретному кейсу
- Модерация: на проверке → одобрен → отклонён
- Избранные отзывы для главной страницы
- Источник отзыва (откуда пришёл)

### 2.6. Услуги и прайс-лист

Каталог услуг компании с ценами и описаниями.

**Возможности:**
- Карточки услуг с иконкой, изображением, описанием
- Гибкие цены: несколько тарифов на разных языках и в разных валютах
- Теги для группировки и поиска
- Публикация/скрытие услуг
- Связь с кейсами и отзывами

### 2.7. Команда / Специалисты

Страница сотрудников компании.

**Возможности:**
- Профили: фото, должность, биография, контакты (email, телефон, LinkedIn, Telegram)
- Привязка к направлениям деятельности
- Порядок сортировки
- Публикация/скрытие профилей

### 2.8. Информация о компании

Направления работы, преимущества, адреса, контакты.

**Возможности:**
- Направления деятельности с иконками и описаниями
- Преимущества компании (конкурентные отличия)
- Адреса офисов с координатами, графиком, телефонами
- Контактные данные: телефон, email, мессенджеры, соцсети

### 2.9. Документы

Загрузка и публикация официальных документов.

**Возможности:**
- Загрузка файлов (PDF, DOC и др.)
- Версионирование документов (дата, номер версии)
- Описание документа на нескольких языках
- Статусы публикации

### 2.10. Часто задаваемые вопросы (FAQ)

Раздел вопросов и ответов.

**Возможности:**
- Вопрос + ответ на нескольких языках
- Категории вопросов
- Управление порядком отображения

### 2.11. Обработка заявок и обращений

Приём, отслеживание и аналитика входящих обращений.

**Возможности:**
- Настраиваемые формы обратной связи (быстрая форма и развёрнутый бриф)
- Статусы обработки: новая → в работе → связались → завершена
- Назначение ответственного сотрудника
- Пометка как спам / отмена
- Фиксация источника обращения (UTM-метки, откуда пришёл, устройство, город)
- Аналитическая сводка: по статусам, источникам, устройствам, дням
- Заметки к обращению
- Защита от спама (ограничение частоты отправки)

### 2.12. Уведомления

Оповещения о событиях по электронной почте и в Telegram.

**Возможности:**
- Уведомление о новой заявке: письмо + Telegram
- Приветственное письмо для новых администраторов
- Письмо для сброса пароля
- Индивидуальные настройки почтового сервера для каждой организации
- Отправка тестовых писем для проверки настроек
- Журнал всех отправленных писем

### 2.13. Telegram-интеграция

Подключение Telegram-бота для мгновенных уведомлений.

**Возможности:**
- Подключение собственного бота организации
- Мгновенные уведомления о новых заявках в чат/группу
- Автоматическая настройка приёма сообщений (webhook)
- Отправка тестового сообщения для проверки

### 2.14. Поисковая оптимизация (SEO)

Инструменты для улучшения видимости сайта в поисковых системах.

**Возможности:**
- Управление заголовками и описаниями страниц для поисковиков
- Автоматическая генерация карты сайта (sitemap) по всем разделам
- Управление файлом robots.txt (что индексировать, а что нет)
- Перенаправления страниц (301, 302) со счётчиком переходов
- Интеграция с IndexNow (мгновенное уведомление поисковиков об обновлениях)
- Файл llms.txt для обнаружения контента AI-ассистентами
- Разметка структурированных данных (JSON-LD)

### 2.15. Файловое хранилище

Централизованное управление медиафайлами.

**Возможности:**
- Загрузка файлов через безопасные ссылки
- Учёт всех загрузок: размер, тип, размеры изображений
- Организация по папкам
- Публичная выдача медиафайлов

### 2.16. Сводная панель (дашборд)

Обзор ключевых показателей в одном окне.

**Возможности:**
- Количество статей, кейсов, услуг, сотрудников, отзывов, FAQ
- Обращения: общее количество, по статусам, за текущий месяц
- Разбивка контента по статусам (опубликовано / черновик / архив)
- Последние действия в системе

### 2.17. Панель владельца платформы

Общая картина по всем организациям.

**Возможности:**
- Сводные показатели: число организаций, пользователей, обращений
- Таблица организаций с метриками
- Детальная статистика по каждой организации
- Тренды: рост организаций, пользователей, обращений во времени
- Система оповещений: неактивные организации, необработанные заявки, проблемные показатели

### 2.18. Экспорт данных

Выгрузка информации для внешнего использования.

**Возможности:**
- Экспорт в CSV и JSON
- Экспорт обращений, команды, SEO-настроек, журнала действий
- Фильтры и выбор колонок

### 2.19. Многоязычность

Поддержка контента на нескольких языках.

**Возможности:**
- Настройка доступных языков для каждой организации
- Все основные разделы поддерживают переводы (статьи, услуги, команда, кейсы, FAQ, документы)
- Указание основного языка
- Поддержка языков с написанием справа налево

## 3. Преимущества для бизнеса

### Что получает организация

1. **Единое место для всей информации** — все данные о компании, контент, обращения, команда собраны в одной системе, а не разбросаны по документам и таблицам
2. **Контроль над публикациями** — система черновиков и статусов не позволяет опубликовать незаконченный материал случайно
3. **Прозрачность обработки заявок** — каждая заявка имеет статус, ответственного и историю; ни одно обращение не теряется
4. **Снижение ручного труда** — уведомления приходят автоматически, карта сайта обновляется сама, SEO-настройки применяются в пару кликов
5. **Аналитика источников обращений** — видно откуда приходят клиенты: из поиска, рекламы, соцсетей; можно оценивать эффективность маркетинга
6. **Гибкость наполнения** — модульная структура: используйте только нужные разделы, остальные отключены
7. **Масштабирование на несколько организаций** — одна система обслуживает десятки клиентов, каждый со своим доменом и данными
8. **Доход от подключения клиентов** — платформа позволяет подключать новые организации и зарабатывать на предоставлении доступа
9. **Разграничение доступов** — маркетолог видит заявки, редактор — контент, админ — всё; каждый работает в своей зоне
10. **Автоматическая SEO-оптимизация** — карты сайта, перенаправления, мета-теги, уведомление поисковиков об обновлениях — всё из коробки
11. **Мгновенные уведомления** — заявка с сайта → за секунды сообщение в Telegram и на почту
12. **Журнал всех изменений** — кто, когда и что менял; полная история для разбора спорных ситуаций
13. **Выгрузка данных** — обращения, команда, журнал действий экспортируются в CSV/JSON для отчётов
14. **Поддержка нескольких языков** — контент на русском, английском и любых других языках для международных компаний
15. **Безопасность данных** — пароли и ключи шифруются, действия логируются, доступ ограничивается по ролям

### Что получают пользователи сайта

1. **Быстрый доступ к информации** — услуги, команда, кейсы, документы структурированы и легко находятся
2. **Удобная отправка обращений** — простые формы с минимумом полей, результат подтверждается
3. **Актуальный контент** — статьи, FAQ, документы обновляются оперативно из единой панели
4. **Доверие через портфолио и отзывы** — реальные кейсы и отзывы с модерацией повышают доверие
5. **Работа на любом устройстве** — контент оптимизирован для просмотра с телефона и компьютера
6. **Быстрая загрузка** — система кэширования снижает время ожидания
7. **Контент на родном языке** — мультиязычность позволяет читать на удобном языке

## 4. Риски и важные моменты

| Область | Описание | Как снижаем |
|---------|----------|-------------|
| **Персональные данные** | Заявки содержат имена, телефоны, email — нужна защита | Шифрование чувствительных данных, разграничение доступов, журнал действий |
| **Модерация контента** | Непроверенный контент может навредить репутации | Система статусов (черновик → публикация), модерация отзывов |
| **Спам-заявки** | Боты могут заваливать формы | Ограничение частоты отправки (3 в минуту), пометка спама |
| **Надёжность работы** | Сбой = потеря обращений и доверия | Проверки здоровья сервисов, автоматический перезапуск, резервные копии |
| **Рост нагрузки** | Больше организаций = больше посетителей | Кэширование, оптимизированные запросы к базе данных, возможность масштабирования |
| **Потеря данных** | Случайное удаление информации | Мягкое удаление (данные не стираются сразу), резервные копии БД |
| **Настройка почты** | Письма могут попадать в спам | Индивидуальная настройка почтового сервера для каждой организации |
| **Доступность доменов** | SSL-сертификаты и привязка доменов | Автоматическое обновление сертификатов, отслеживание статуса |

### Ключевые технические решения

- **Stack:** FastAPI + SQLAlchemy 2.0 (async) + PostgreSQL 16 + Redis 7
- **Auth:** JWT (access 30min + refresh 7d), bcrypt, token blacklist в Redis
- **Multi-tenancy:** tenant_id FK на всех таблицах, middleware для tenant resolution
- **File Storage:** S3-compatible (MinIO dev / Selectel prod), presigned URL upload
- **Email:** aiosmtplib (SMTP), httpx (SendGrid/Mailgun), per-tenant config, Fernet encryption
- **Migrations:** Alembic (24 миграции), autogenerate + ручные правки
- **Middleware:** Rate limiting (Redis sliding window), Cache-Control/ETag, CORS, Request logging
- **Content Blocks:** Полиморфная связь ContentBlock → Article/Case/Service/Employee
- **SEO:** Сегментированный sitemap, ETag caching, IndexNow push, llms.txt generation
- **Audit:** Immutable AuditLog с JSONB changes (before/after)
- **Soft Delete:** Все контентные сущности, SoftDeleteMixin
- **Optimistic Locking:** VersionMixin с auto-increment

### Ключевые процессы (workflow)

1. **Публикация контента:** draft → published (set published_at) → archived
2. **Модерация отзывов:** pending → approved / rejected
3. **Обработка заявки:** new → in_progress → contacted → completed / spam / cancelled
4. **Приглашение пользователя:** create → welcome email → force_password_change → active
5. **Подключение домена:** add domain → pending SSL → active SSL / error

### Неочевидные зависимости и вопросы к клиенту

1. **Платежи/взносы:** В текущей реализации нет модуля оплаты. Планируется ли приём платежей через сайт? Если да — нужна интеграция (ЮKassa, Stripe и др.) — оценка отдельно.
2. **Личный кабинет посетителя:** Сейчас есть только панель администратора. Нужен ли личный кабинет для посетителей сайта (регистрация, история обращений)?
3. **Поиск по сайту:** Полнотекстовый поиск не реализован. Нужен ли? (PostgreSQL FTS или Elasticsearch)
4. **Комментарии к статьям:** Не реализованы. Планируется ли?
5. **Уведомления пользователей:** Push-уведомления в браузере? Email-рассылки (не транзакционные)?
6. **Интеграция с CRM:** Нужна ли передача заявок в amoCRM, Bitrix24 и т.д.?
7. **Импорт данных:** Есть экспорт, но нет импорта. Нужна ли миграция данных из текущей системы?
8. **Политика хранения данных:** Как долго хранить заявки, логи, аудит?
9. **Подписки/рассылки:** Нужна ли подписка на новости (email-дайджест)?
10. **Мобильное приложение:** Планируется ли? Текущий API полностью готов для мобильных клиентов.
11. **Социальные сети:** Нужна ли автопубликация в соцсети при публикации статьи?
12. **Лимиты по тарифам:** Ограничения для организаций (макс. пользователей, макс. статей)?
13. **Резервное копирование:** Какая политика? Ежедневно? Хранить 30 дней?
14. **SLA / Доступность:** Какие требования к uptime? 99.9%?
15. **GDPR/ФЗ-152:** Нужны ли специальные механизмы (удаление персональных данных по запросу, согласие на обработку)?
